📌 1. What is Git?

  • Track changes to files
  • Revert to earlier versions
  • Collaborate on code

Core Git Concepts

  • Repository: Folder tracked by Git
  • Commit: A saved version
  • Branch: Parallel workspace
  • Staging Area: Where files are prepared

🌐 2. What is GitHub?

Cloud platform for hosting Git repositories with features like pull requests, issues, and collaboration tools.

🧰 3. Git Setup & Basic Commands

Initialize Repository

git init

Add files

git add .

Commit files

git commit -m "Initial commit"

Status

git status

View history

git log

🔗 4. Connect to GitHub

git remote add origin https://github.com/user/repo.git
git branch -M main
git push -u origin main

Pull latest from GitHub

git pull origin main

📁 5. Cloning a Repo

git clone https://github.com/user/repo.git

🙈 6. .gitignore

not_push.html
node_modules/
*.log

Remove tracked file

git rm --cached not_push.html

🔀 7. Branching

git branch new-feature       # create
git checkout new-feature    # switch
# or
git checkout -b new-feature # both

Merge to main

git checkout main
git merge new-feature

Delete Branch

git branch -d new-feature
git push origin --delete new-feature

⚠️ Mistakes & Fixes

❌ Tracked a file by mistake

git rm --cached filename

❌ Used 'touch' on Windows

New-Item .gitignore

📋 Cheatsheet

Command Purpose
git init Start a new local repo
git status Check file changes
git add . Stage all files
git commit -m "msg" Save snapshot
git log View history
git remote add origin Link with GitHub
git push -u origin main Push changes
git pull origin main Sync updates
git checkout -b feature Create & switch branch
git merge feature Merge into current
git branch -d feature Delete local branch
git clone URL Download GitHub repo
